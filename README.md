# 💳 Electro-Wallet API

Современный бэкенд-сервис для управления электронным кошельком. Проект построен на передовом стеке технологий и демонстрирует реализацию безопасных транзакций, ролевой модели доступа и чистого кода.

## 🚀 Стек технологий

* **Java 25** (Preview features)
* **Spring Boot 4.0.3**
* **Spring Security** (Basic Auth & Password Hashing)
* **PostgreSQL** (Database)
* **Docker & Docker Compose** (Containerization)
* **MapStruct** (Object mapping)
* **Lombok** (Boilerplate reduction)
* **JUnit 5 & Mockito** (Unit testing)

## 🛠 Функционал

### Пользователь (User):
* **Регистрация и аутентификация:** Безопасный вход через Basic Auth.
* **Переводы:** Быстрые денежные переводы по номеру телефона.
* **История:** Просмотр списка своих транзакций и деталей конкретной операции.

### Администратор (Admin):
* **Глобальный аудит:** Просмотр всех транзакций в системе.
* **Управление пользователями:** Возможность блокировки и разблокировки аккаунтов.
* **Контроль:** Доступ к детальной информации по любой транзакции любого пользователя.

## 🏗 Архитектура проекта

Проект реализован с соблюдением принципов чистой архитектуры:
* **Layered Architecture:** Разделение ответственности (Controller -> Service -> Repository).
* **DTO Pattern:** Использование Request/Response объектов для безопасности и гибкости API.
* **Global Exception Handling:** Централизованная обработка ошибок (`@RestControllerAdvice`) с использованием кастомных исключений и констант `ErrorMessages`.
* **Logging:** Логирование ключевых событий (вход, переводы, блокировки) для аудита.

## 🐳 Запуск через Docker

Для запуска проекта одной командой (база + приложение) убедитесь, что у вас установлен Docker Desktop.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone [https://github.com/ваш-логин/electro_wallet.git](https://github.com/ваш-логин/electro_wallet.git)
    cd electro_wallet
    ```

2.  **Запустите проект:**
    ```bash
    docker-compose up --build
    ```

Приложение будет доступно по адресу: `http://localhost:8080`

## 🧪 Тестирование и валидация

* **Unit Tests:** Бизнес-логика сервисов протестирована с использованием **Mockito**.
* **Validation:** Входящие данные проверяются через `spring-boot-starter-validation` (номера телефонов, суммы, пустые поля).
* **Postman:** Все эндпоинты успешно протестированы вручную.
