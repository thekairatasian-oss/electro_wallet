# Electro Wallet Service 💳📱

Современный бэкенд-сервис для электронного кошелька с банковской логикой, поддержкой ролей, транзакционным аудитом и контейнеризацией через Docker. Проект разработан по стандартам enterprise-разработки.

---

## 🛠 Технологический стек

* **Язык разработки:** Java 21 / 25
* **Фреймворк:** Spring Boot 3.x (Spring Web, Spring Security, Spring Data JPA)
* **База данных:** PostgreSQL
* **Маппинг и Бойлерплейт:** MapStruct, Lombok
* **Безопасность:** Spring Security (Ролевая модель, Basic Auth)
* **Контейнеризация:** Docker, Docker Compose (Multi-stage build)
* **Инструменты сборки и миграций:** Maven

---

## 🚀 Архитектура и Ключевой Функционал

Проект построен на базе **многослойной архитектуры (Layered Architecture)** с четким разделением ответственности (Controller -> Service -> Repository) и использованием DTO для передачи данных.

### 1. Бизнес-логика и Финтех операции
* **Управление кошельками:** Привязка аккаунта к уникальному номеру телефона пользователя.
* **Атомарные транзакции:** Переводы между кошельками реализованы с гарантией целостности данных. При сбоях происходит полный откат изменений (Rollback).
* **Безопасность баланса:** Защита от состояния гонки (Race Condition) и одновременных списаний с помощью механизмов блокировок (Optimistic Locking).
* **Пополнение счета:** Эндпоинты для безопасного депозита средств.

### 2. Безопасность и Ролевая модель (RBAC)
Доступ к эндпоинтам разграничен на основе ролей пользователей:
* **USER (Пользователь):** Может регистрироваться в системе, просматривать свой баланс и историю личных транзакций.
* **ADMIN (Администратор):** Обладает полными правами. Может просматривать всех пользователей системы, видеть общую историю транзакций, а также блокировать/удалять учетные записи.

### 3. Обработка ошибок и Логирование
* **Global Exception Handling:** Реализован централизованный обработчик ошибок (`@ControllerAdvice`), который возвращает клиенту понятные и стандартизированные JSON-ответы при возникновении бизнес-исключений.
* **Логирование:** Все ключевые финансовые операции и изменения статусов пользователей логируются с помощью фасада `SLF4J`.

## 📦 Развертывание и запуск приложения через Docker

В проект внедрена **многоэтапная сборка (Multi-stage build)**. Вам не нужно устанавливать Java или PostgreSQL на свой компьютер — всё разворачивается одной командой.

### Предварительные требования
* Установленный **Docker** и **Docker Compose**.
